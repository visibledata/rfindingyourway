Finding youR way

R is not software.
It's a scripting language, a vast collection of tools and a vibrant user community.

You can't master R
But you can become fluent in using it.

Code

Before you can feel confident reading or writing R code, there are some inescapable things you need to learn about the language and its syntax.

  • Brackets
  • snake_case vs CamelCase
  • Named and default arguments

Check out our R Code 101 page

Workflow

Often the most tricky part of becoming confident in using R is getting used to the workflow, including;

  • Script files, the console and prompt
  • Finding, installing and using packages
  • RStudio projects

Check out our R Workflows 101 page

Data

Most folks are using R to clean, wrangle or otherwise trying to get data to behave. You'll need to understand:

  • Rectangular data structures
  • Subsetting, filtering and indexing data
  • Importing data from .csv, Excel, SPSS, SAS, databases...

Check out our R Data 101 page

Communicate

R has incredible tools for visualising your data, models and analyses. These include the following tools:

  • Static dataviz with ggplot2
  • Create PDF, HTML, Word reports with RMarkdown
  • Interactive dashboards with Shiny

Check out our R Communicate 101 page

Program

Many businesses rely on complicated Excel-based workflows for compiling scheduled reports. These workflows almost always involve significant manual effort because the data-containing Excel workbooks change frequently.

Starting to use a scripting language for your data-based workflows takes a little getting used to. But once you've got the basic concepts under your belt you'll find significant time savings, reduced manual interventions and the ability to programmatically test outputs are valid.

There are three fundamental things you need to learn before you can successfully solve your problems programmatically.

  • Designing user-defined functions
  • Repeating actions with loops and/or maps
  • Handling errors, missing data and NAs
